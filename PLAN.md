# 작업 계획서 — Blender 3D 캐릭터 리깅 & 애니메이션 파이프라인 (for Unity)

> 작성일: 2026-07-17 · 상태: **Phase 0 착수 전**
> 이 문서는 작업 진행에 따라 계속 갱신한다. 각 Phase의 체크박스가 진행 상황의 기준이다.

## 1. 목표

게임 제작용 3D 캐릭터의 **리깅 → 애니메이션 → Unity 반입**까지를 에이전트(Claude Code)가
자동화 스크립트 기반으로 수행하는 파이프라인을 구축한다.

- 캐릭터 모델은 사용자가 직접 제작하여 제공한다.
- 애니메이션은 **키 포즈 이미지 레퍼런스** 기반 수제 키프레임 + **Mixamo 리타게팅** 하이브리드로 제작한다.
- 최종 산출물은 Unity **Humanoid 아바타 규격**을 만족하는 FBX.

## 2. 확정된 기술 방침

| 항목 | 결정 | 이유 |
|---|---|---|
| 타깃 엔진 | Unity (Humanoid) | 리타게팅 호환성 — Mixamo 등 외부 애니메이션을 그대로 활용 가능 |
| 작업 방식 | **bpy 스크립트가 본체, MCP는 보조** | 반복 작업(재리깅·재익스포트)이 "재실행"으로 끝나야 함. 재현성·버전관리 확보 |
| MCP 용도 | 인터랙티브 확인/미세조정 전용 | 뷰포트 보며 실시간 보정할 때만 사용. 확정된 절차는 반드시 스크립트로 환원 |
| 리깅 | Rigify 메타리그 → 컨트롤 리그 자동 생성 | Blender 표준. 체형 맞춤 후 자동 생성으로 IK/FK 확보 |
| 익스포트 | 컨트롤 리그 → **deform 본만 베이크** → FBX | Rigify 컨트롤 본 수백 개는 엔진에서 사용 불가 |
| 검증 | 렌더 프리뷰를 에이전트가 직접 읽고 교정 | 이미지 레퍼런스와 비교하며 self-correction 루프 |
| Blender | 5.1 (로컬 설치, `C:\Program Files\Blender Foundation\Blender 5.1`) | headless 실행: `blender --background --python <script>` |

## 3. 저장소 구조

```
anime/
├── PLAN.md          # 이 문서
├── scripts/         # bpy 자동화 스크립트 (파이프라인 본체)
│   ├── rig/         #   리깅 관련 (메타리그 적용, 웨이트)
│   ├── anim/        #   애니메이션 (포즈 적용, 베이크)
│   └── export/      #   Unity용 FBX 익스포트, 프리뷰 렌더
├── assets/          # .blend 소스 파일 (캐릭터, 템플릿)
├── refs/            # 키 포즈 레퍼런스 이미지 (사용자 제공)
├── exports/         # 산출물 (.fbx) — Unity가 바라보는 폴더
└── previews/        # 검증용 렌더 이미지 (git 제외 고려)
```

## 4. Phase별 계획

### Phase 0 — 파이프라인 검증 (캐릭터 없이 즉시 가능) ✅ 완료

더미 마네킹으로 전 구간을 뚫어, 이후 문제 발생 시 변수를 "캐릭터"로 한정한다.
Blender→Unity의 알려진 함정(축 방향, 스케일 100배, 본 롤)을 여기서 해소한다.

- [x] 저장소 폴더 구조 생성, headless Blender 실행 래퍼 스크립트
- [x] 더미 휴머노이드 메시 생성 스크립트
- [x] Rigify 메타리그 적용 → 컨트롤 리그 생성 → 자동 웨이트
- [x] 간단한 idle 애니메이션 1개 (절차적 생성으로 충분)
- [x] deform 본 베이크 → Unity Humanoid 규격 본 네이밍 → FBX 익스포트
- [x] 턴테이블/다각도 프리뷰 렌더 스크립트 (에이전트 검증 루프의 눈)
- [x] **완료 기준**: 익스포트된 FBX를 Unity에서 임포트했을 때 Humanoid 아바타로
      오류 없이 인식되고 idle이 재생됨 (사용자 확인)
      (자동 검증 8/8 통과 2026-07-17. 사용자 조건부 승인 2026-07-18 — "Phase 0 치고는 괜찮음".
      최종 눈검증은 실제 캐릭터 반입·보완 후 재수행하기로 함)

### Phase 1 — 실제 캐릭터 리깅 (캐릭터 수령 후)

**착수 전 필수 게이트: L/R 스왑 규명** — bone_map.py의 좌우 이름 스왑이 04_export.py의
사전 회전 트릭과 결합된 보상인지 비대칭 더미로 판별하고, 가능하면 스왑 없는 올바른
익스포트 설정으로 대체한다. (대칭 더미에서는 스왑 오류가 보이지 않음)

- [ ] 캐릭터 .blend/.fbx 수령 → `assets/` 반입, 스케일·축·N-gon 등 사전 점검 스크립트
- [ ] 메타리그를 캐릭터 체형에 맞춤 (스크립트 + MCP 인터랙티브 병용)
- [ ] 자동 웨이트 → 문제 부위(어깨, 겨드랑이, 치마/머리카락 등) 보정
- [ ] 다각도 포즈 테스트 렌더로 웨이트 품질 검증
- [ ] **완료 기준**: 극단 포즈(팔 들기, 쪼그리기)에서 메시 파손 없음

### Phase 2 — 애니메이션 제작 (프로젝트 본체)

**2a. Mixamo 리타게팅 (이동 계열: idle / walk / run 등)**
- [ ] Mixamo 애니메이션 → 우리 리그로 리타게팅하는 스크립트
- [ ] Action 단위 관리 + NLA 정리 규칙 수립

**2b. 이미지 레퍼런스 수제 키프레임 (개성 액션)**
- [ ] 레퍼런스 반입 규칙: 키 포즈 4~8장 + 타이밍 정보, `refs/<action명>/`에 배치
- [ ] 포즈 → 본 회전 적용 → 렌더 → 레퍼런스와 나란히 비교 → 교정 루프
- [ ] 측면 뷰 우선, 깊이 모호한 동작은 2~3각도 레퍼런스 요청
- [ ] **완료 기준**: 사용자가 프리뷰 렌더를 보고 액션별 승인

### Phase 3 — Unity 반입 자동화

- [ ] "스크립트 한 방"으로 전체 Action → `exports/` FBX 일괄 갱신
- [ ] 익스포트 설정 고정 (스케일, 축, 베이크 옵션) 및 회귀 체크
- [ ] (선택) Unity 프로젝트 측 임포트 설정 가이드 문서화

## 5. 협업 규칙 (사용자 ↔ 에이전트)

- **캐릭터/레퍼런스 제공**: 사용자가 `assets/`, `refs/`에 파일을 넣고 알려주면 에이전트가 반입 점검부터 수행
- **레퍼런스 이미지 형식**: 액션당 키 포즈 4~8장(전 프레임 불필요), 가능하면 측면 뷰,
  스프라이트 시트면 순서가 곧 타이밍. 전체 시트가 있으면 검증용 정답지로 활용
- **AI 생성 레퍼런스 사용 시**: 포즈만 참고, 신체 비율은 3D 모델 기준 고정 (프레임 간 비율 흔들림 주의)
- **승인 게이트**: 각 Phase 완료 기준 중 "사용자 확인" 항목은 프리뷰 렌더 기반으로 사용자가 판단
