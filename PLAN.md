# 작업 계획서 — Blender 3D 캐릭터 리깅 & 애니메이션 파이프라인 (for Unity)

> 작성일: 2026-07-17 · 상태: **Phase 0 착수 전**
> 이 문서는 작업 진행에 따라 계속 갱신한다. 각 Phase의 체크박스가 진행 상황의 기준이다.

## 1. 목표

게임 제작용 3D 캐릭터의 **리깅 → 애니메이션 → Unity 반입**까지를 에이전트(Claude Code)가
자동화 스크립트 기반으로 수행하는 파이프라인을 구축한다.

- 캐릭터 모델은 사용자가 직접 제작하여 제공한다.
- 애니메이션은 **키 포즈 이미지 레퍼런스** 기반 수제 키프레임 + **Mixamo 리타게팅** 하이브리드로 제작한다.
- 최종 산출물은 Unity **Humanoid 아바타 규격**을 만족하는 FBX.

## 2. 확정된 기술 방침

| 항목 | 결정 | 이유 |
|---|---|---|
| 타깃 엔진 | Unity (Humanoid) | 리타게팅 호환성 — Mixamo 등 외부 애니메이션을 그대로 활용 가능 |
| 작업 방식 | **bpy 스크립트가 본체, MCP는 보조** | 반복 작업(재리깅·재익스포트)이 "재실행"으로 끝나야 함. 재현성·버전관리 확보 |
| MCP 용도 | 인터랙티브 확인/미세조정 전용 | 뷰포트 보며 실시간 보정할 때만 사용. 확정된 절차는 반드시 스크립트로 환원 |
| 리깅 | Rigify 메타리그 → 컨트롤 리그 자동 생성 | Blender 표준. 체형 맞춤 후 자동 생성으로 IK/FK 확보 |
| 익스포트 | 컨트롤 리그 → **deform 본만 베이크** → FBX | Rigify 컨트롤 본 수백 개는 엔진에서 사용 불가 |
| 검증 | 렌더 프리뷰를 에이전트가 직접 읽고 교정 | 이미지 레퍼런스와 비교하며 self-correction 루프 |
| Blender | 5.1 (로컬 설치, `C:\Program Files\Blender Foundation\Blender 5.1`) | headless 실행: `blender --background --python <script>` |

## 3. 저장소 구조

```
anime/
├── PLAN.md          # 이 문서
├── scripts/         # bpy 자동화 스크립트 (파이프라인 본체)
│   ├── rig/         #   리깅 관련 (메타리그 적용, 웨이트)
│   ├── anim/        #   애니메이션 (포즈 적용, 베이크)
│   └── export/      #   Unity용 FBX 익스포트, 프리뷰 렌더
├── assets/          # .blend 소스 파일 (캐릭터, 템플릿)
├── refs/            # 키 포즈 레퍼런스 이미지 (사용자 제공)
├── exports/         # 산출물 (.fbx) — Unity가 바라보는 폴더
└── previews/        # 검증용 렌더 이미지 (git 제외 고려)
```

## 5. 협업 규칙 (사용자 ↔ 에이전트)

- **캐릭터/레퍼런스 제공**: 사용자가 `assets/`, `refs/`에 파일을 넣고 알려주면 에이전트가 반입 점검부터 수행
- **레퍼런스 이미지 형식**: 액션당 키 포즈 4~8장(전 프레임 불필요), 가능하면 측면 뷰,
  스프라이트 시트면 순서가 곧 타이밍. 전체 시트가 있으면 검증용 정답지로 활용
- **AI 생성 레퍼런스 사용 시**: 포즈만 참고, 신체 비율은 3D 모델 기준 고정 (프레임 간 비율 흔들림 주의)
- **승인 게이트**: 각 Phase 완료 기준 중 "사용자 확인" 항목은 프리뷰 렌더 기반으로 사용자가 판단
